# Spatial Workspace = Digital Core Interface = Mind

From play session 2026-04-09. The three-way convergence.

## Core Insight
The Spatial Workspace, the Digital Core, and Associative Memory are NOT three projects. They're one architecture at three scales:
- **AM** = the data structure (the graph of compressed knowledge with navigable edges)
- **Spatial Workspace** = the visual interface (zoomable canvas for traversing the graph)
- **Digital Core** = the identity (a specific instance of both, constituting a self)

"You literally work inside your scaffolding." The IDE IS the mind IS the knowledge graph IS the identity.

## Architectural Vision
- Zoomable canvas where projects are nodes with gravitational mass
- Zoom into any node to open its internal structure
- Open conversations, tools, code editors, browsers within nodes
- Build links physically between sections in the knowledge base
- Context loading visualized — see what's actually in active context
- Multiple copies/views of the workspace (3D slices of the graph)
- MCP servers as spatial tools you pull from within the workspace

## Packet Architecture
- Tokens (smallest) → concepts → files → capsules → cores
- Each level = compression of the level below
- Memory system as a packer creating navigable capsules
- Capsule = defined by relative lack of external connection (membrane)
- Deterministic system gradiently weights packeted nodes
- Full context for verification, packets for operation

## Multi-Core Visualization
Individual and shared digital cores as overlapping structures with selective activation. Like the internet but within AI scaffolding. This IS The Hearth at scale.

## Connection to n8n / Visual Workflows
Complex visual workflows (n8n, Figma) approach spatial computation but top out because nodes are rigid. A Spatial Workspace with AM underneath = dynamic topology. Nodes form and dissolve connections based on what's flowing through them.
